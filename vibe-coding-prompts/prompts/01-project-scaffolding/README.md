# 🏗️ Project Scaffolding Prompts

Use these prompts when starting a new project from scratch.

---

## 1. Full Stack Project Scaffolder

**Best for:** Any new web app idea  
**Works with:** Claude, ChatGPT, Cursor

```
I want to build [PROJECT DESCRIPTION].

Tech stack: [e.g. Next.js 14, Tailwind CSS, Supabase, TypeScript]

Please:
1. Suggest the complete folder structure
2. List all dependencies I need to install with exact versions
3. Create the main entry files with boilerplate code
4. Identify the 3 most important components to build first
5. Point out any potential technical challenges early

Keep it production-ready and beginner-friendly.
```

---

## 2. MVP Feature Planner

**Best for:** Deciding what to build first  
**Works with:** Claude, ChatGPT

```
I'm building [APP NAME]. It should do [CORE FUNCTION].

My target user is: [DESCRIBE USER]
My deadline is: [TIMEFRAME]

Help me define my MVP by:
1. Listing only the features absolutely needed for launch (max 5)
2. Identifying what I should NOT build yet
3. Suggesting the build order
4. Estimating complexity (easy/medium/hard) for each feature

Be ruthless about cutting scope.
```

---

## 3. Tech Stack Advisor

**Best for:** Choosing the right tools  
**Works with:** Claude, ChatGPT

```
I want to build [PROJECT TYPE]. Here's my situation:

- My skill level: [beginner/intermediate]
- Budget: [free/low-cost/flexible]
- Expected users: [number range]
- Must-have features: [list]
- Timeline: [timeframe]

Recommend the best tech stack for this and explain WHY each tool fits my situation. Also tell me what NOT to use and why.
```

---

## 4. Database Schema Designer

**Best for:** Planning your data models  
**Works with:** Claude, ChatGPT

```
I'm building [APP NAME] with these features: [LIST FEATURES]

Design a database schema for me:
1. List all tables/collections needed
2. Define fields and data types for each
3. Show relationships between tables
4. Suggest indexes for performance
5. Flag any data I should NOT store (privacy/security)

Use [PostgreSQL / MongoDB / Supabase] syntax.
```

---

## 5. Project README Generator

**Best for:** Documenting your project  
**Works with:** Claude, ChatGPT

```
Generate a professional README.md for my project:

- Project name: [NAME]
- What it does: [DESCRIPTION]
- Tech stack: [LIST]
- How to install: [STEPS]
- Key features: [LIST]
- Target audience: [WHO]

Make it look great on GitHub with proper badges, sections, and formatting. Include a contributing section and license.
```

---

## 6. Environment Setup Checklist

**Best for:** Setting up a new machine or project  
**Works with:** Claude, ChatGPT, Cursor

```
I'm setting up a [Next.js / React / Node] project on [Mac/Windows/Linux].

Generate a complete setup checklist:
1. All tools to install (with versions)
2. Environment variables needed (.env.example)
3. Git configuration steps
4. VS Code extensions to install
5. Common gotchas to avoid for this stack

Format it as a step-by-step checklist I can follow.
```
