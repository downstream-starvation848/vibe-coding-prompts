# Contributing to vibe-coding-prompts

First off — thank you! This repo exists because the community shares what works.

---

## What Makes a Good Prompt?

A prompt belongs in this repo if it:

- ✅ You've personally used it and got a great result
- ✅ It's reusable — not specific to just your project
- ✅ It uses `[BRACKETED PLACEHOLDERS]` for variable parts
- ✅ It produces better results than a basic ask would

A prompt does NOT belong here if it:

- ❌ Only works for one specific use case with no flexibility
- ❌ Is basically just a question, not a structured prompt
- ❌ Duplicates an existing prompt without meaningful improvement

---

## How to Contribute

### Option 1: Add a New Prompt

1. Fork this repo
2. Find the right category folder in `prompts/`
3. Add your prompt to the relevant `README.md`
4. Follow the existing format:
   - Prompt name
   - **Best for:** one-liner
   - **Works with:** tool names
   - The prompt in a code block
5. Open a Pull Request with a clear title like `Add: [prompt name] to [category]`

### Option 2: Improve an Existing Prompt

Found a way to get better results from an existing prompt? Open a PR with:
- The original prompt
- Your improved version
- A brief note on what changed and why it's better

### Option 3: Add a Template

Templates go in the `templates/` folder. Each template should have:
- Its own folder
- A `README.md` explaining what it is and a prompt to generate it
- The actual template code (if applicable)

---

## Pull Request Guidelines

- Keep PRs focused — one prompt or improvement per PR
- Test your prompt before submitting
- Be respectful in reviews — we're all learning here

---

## Found a Bug or Have an Idea?

Open an issue! Label it:
- `bug` — something is wrong
- `enhancement` — idea for improvement
- `new-prompt` — suggesting a prompt you don't have time to write yet

---

## Code of Conduct

Be kind. This is a learning community. Everyone here is figuring things out.
